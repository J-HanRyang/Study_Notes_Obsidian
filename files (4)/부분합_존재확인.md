# 부분합 존재 확인

## 문제 요약
연속된 수들의 합이 T가 되는 경우가 있으면 1, 없으면 0 출력

## 풀이 접근
투 포인터로 구간 합 유지 — 정렬 하면 안 됨!

## 코드
```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int N, T;
    scanf("%d %d", &N, &T);

    int *arr = (int*)malloc(sizeof(int) * N);
    for (int i = 0; i < N; i++) {
        scanf("%d", &arr[i]);
    }

    int left = 0;
    int right = 0;
    int sum = arr[0];

    while (right < N) {
        if (sum == T) break;
        else if (sum < T) {
            if (right + 1 < N) {
                right++;
                sum += arr[right];
            } else break;
        }
        else {
            sum -= arr[left];
            left++;
        }
    }

    if (sum == T) printf("1");
    else printf("0");

    free(arr);
    arr = NULL;

    return 0;
}
```

## 핵심 포인트
- 연속 구간 문제 → 정렬 금지
- sum은 루프 밖에서 선언
- right++ 전에 범위 체크 (right + 1 < N)

## 실수 / 막혔던 것
- sum을 루프 안에서 선언해서 누적 안 됨
- 범위 체크 없이 right++ 해서 오버플로우

# 두 수의 합

## 문제 요약
정렬된 배열에서 합이 T인 쌍이 있으면 1, 없으면 0 출력

## 풀이 접근
이중 for문 O(N²) → 시간초과
투 포인터 O(N) 으로 해결

## 코드
```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int N, T;
    scanf("%d %d", &N, &T);

    int *arr = (int*)malloc(sizeof(int) * N);
    for (int i = 0; i < N; i++) {
        scanf("%d", &arr[i]);
    }

    int left = 0;
    int right = N - 1;

    while (left < right) {
        int sum = arr[left] + arr[right];
        if (sum == T) break;
        else if (sum > T) right--;
        else left++;
    }

    if (arr[left] + arr[right] == T) printf("1");
    else printf("0");

    free(arr);
    arr = NULL;

    return 0;
}
```

## 핵심 포인트
- left, right는 인덱스, 값 아님
- 이미 정렬된 배열이라 qsort 불필요

## 실수 / 막혔던 것
- left, right를 값으로 착각

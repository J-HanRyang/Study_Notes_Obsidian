## 문제 요약
N개의 문서가 우선순위 큐에 있을 때, M번째 문서가 몇 번째로 인쇄되는지 출력

## 풀이 접근
구조체로 우선순위와 인덱스를 같이 추적. 현재 문서보다 높은 우선순위가 뒤에 있으면 큐 뒤로 재삽입, 없으면 인쇄 카운트 증가.

## 코드
```c
#include <stdio.h>

int main() {
    typedef struct {
        int priority;
        int index;
    } Job;

    Job queue[1000];
    int front = 0, rear = 0;

    int N, M;
    scanf("%d %d", &N, &M);

    for (int i = 0; i < N; i++) {
        scanf("%d", &queue[rear].priority);
        queue[rear].index = i;
        rear++;
    }

    int count = 0;

    while (front < rear) {
        Job cur = queue[front++];

        int hasHigher = 0;
        for (int i = front; i < rear; i++) {
            if (queue[i].priority > cur.priority) {
                hasHigher = 1;
                break;
            }
        }

        if (hasHigher) {
            queue[rear++] = cur;
        } else {
            count++;
            if (cur.index == M) {
                printf("%d", count);
                return 0;
            }
        }
    }

    return 0;
}
```

## 핵심 포인트
- 구조체로 우선순위 + 인덱스 같이 추적해야 M번째 문서 식별 가능
- 구조체 비교는 반드시 멤버로 접근 (`cur.priority > queue[i].priority`)
- 큐 배열 크기를 여유 있게 잡아야 함 (재삽입으로 rear가 N 이상 증가)

## 실수 / 막혔던 것
- `cur < queue[i]` 처럼 구조체 통째로 비교하면 컴파일 에러
- `scanf("%d", queue[i])` → `scanf("%d", &queue[i].priority)` 로 멤버 접근 필요
- 루프 내에서 `queue[front]` 대신 `queue[i]` 써야 함
